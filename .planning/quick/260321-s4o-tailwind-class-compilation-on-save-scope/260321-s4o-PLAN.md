  <action>
Create `apps/starter/src/hooks/compileBlockStyles.ts` as a `CollectionAfterChangeHook`:

1. The hook walks the document's block tree (doc.layout, and recursively into blocks that have `children` arrays like Container/Grid).

2. For each block, extract `styles.customCSS.classes` (string of Tailwind class names) and `styles.customCSS.inlineCSS` (raw CSS declarations).

3. Build a `compiledCSS` string per block:
   - **Tailwind classes:** Use the Tailwind v4 programmatic API. Call `compile()` once with a minimal CSS input (`@import "tailwindcss";`) to get a compiler, then call `build(candidates)` with the collected class names. This generates the CSS definitions those classes need. Wrap the output so it's present on the page.
   - **Scoped inline CSS:** If `inlineCSS` exists, wrap it as `#block-{block.id} { ...inlineCSS... }`. This is the Shopify pattern -- style tag scoping instead of inline style attributes.
   - Combine both into `compiledCSS` string on the block.

4. Important: The `compile()` call should happen ONCE per document save (not per block). Collect ALL Tailwind class names from ALL blocks, call `build()` once, then store the full compiled CSS output as a single `_compiledTailwindCSS` string on the document root (not per-block). The per-block scoped CSS (`#block-{id} { inlineCSS }`) should be collected into a single `_compiledScopedCSS` string on the doc root too. This avoids needing hidden fields on every block.

   Actually, simpler approach: store a single `_compiledBlockCSS` field at the doc root level that contains ALL compiled output (tailwind definitions + all scoped block CSS). This is one string rendered as one style tag.

5. For the Tailwind compile call, use:
   ```typescript
   import { compile } from 'tailwindcss'
   const compiler = await compile('@import "tailwindcss";')
   const tailwindCSS = compiler.build(allClassNames)
   ```
   Where `allClassNames` is the deduplicated array of all Tailwind classes from all blocks.

6. The hook should update the document via `req.payload.update()` with `_compiledBlockCSS` set, using `context: { disableRevalidate: true, disableCompileStyles: true }` to prevent infinite loops (check `context.disableCompileStyles` at the top of the hook to bail early).

7. Register this hook in Pages.ts, Posts.ts, and TemplateParts.ts by adding it to their `afterChange` arrays alongside existing hooks. Also add a `_compiledBlockCSS` hidden text field to each collection's fields array:
   ```typescript
   { name: '_compiledBlockCSS', type: 'textarea', admin: { hidden: true } }
   ```

NOTE: The `compile()` call needs the CSS input to resolve `@import "tailwindcss"`. Pass `{ loadStylesheet }` option or use a simpler base CSS like `@theme {}` with just the default theme. If `@import "tailwindcss"` doesn't resolve in Node context, use the project's actual globals.css path via `{ loadStylesheet }` from `@tailwindcss/node` or just use a base CSS string that includes the default theme inline. Test which approach works. If `@tailwindcss/node` is needed, install it as a dev dependency (it provides `loadStylesheet` for filesystem resolution). Alternatively, `compile('@tailwind base;')` or just `compile('')` may work for v4 -- the key is getting a working `build()` function that knows Tailwind's utility classes. Check if `compile('@import "tailwindcss";')` works by itself in v4.2.2 -- it may need the `loadStylesheet` option to resolve the import. If so, use:
   ```typescript
   import { compile } from 'tailwindcss'
   import { Scanner, loadStylesheet } from '@tailwindcss/node'
   const compiler = await compile('@import "tailwindcss";', { loadStylesheet })
   ```
   And add `@tailwindcss/node` to dependencies. But first try without it -- if `compile` can work with inline theme definitions instead of imports, that's simpler.

Simplest reliable approach: Read the project's globals.css content as a string and pass it to `compile()`. This ensures the compiler has the full theme. Use `fs.readFileSync(path.join(process.cwd(), 'src/app/(frontend)/globals.css'), 'utf-8')` to read it. This avoids needing `@tailwindcss/node` entirely.
