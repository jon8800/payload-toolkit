  <name>Task 1: Create template parts resolution utility and middleware</name>
  <files>src/utilities/resolveTemplateParts.ts, src/middleware.ts</files>
  <action>
1. Create `src/utilities/resolveTemplateParts.ts`:
   - Export async function `resolveTemplateParts(partType: 'header' | 'footer' | 'custom', currentPath: string, currentCollection?: string)`
   - Use `getPayload({ config: configPromise })` to query template-parts collection
   - Query all published template parts of the given type: `where: { type: { equals: partType }, _status: { equals: 'published' } }`, `limit: 0` (template parts are few, fetch all)
   - Implement priority resolution (most specific wins):
     1. `specificPages` mode: match if `currentPath` matches any of the part's `pages` relationship slugs (check both `/{slug}` and slug directly)
     2. `collectionType` mode: match if `currentCollection` equals the part's `collectionType` value
     3. `excludePages` mode: match if `currentPath` does NOT match any of the part's `pages` slugs
     4. `entireSite` mode: fallback, always matches
   - Return the matched template part document (with layout blocks) or null
   - Wrap with React `cache()` for request-level deduplication
   - Use `depth: 2` on the find query to ensure layout blocks are fully populated

2. Create `src/middleware.ts`:
   - Export middleware function that sets `x-pathname` header on the response using `request.nextUrl.pathname`
   - Export config with matcher: `['/((?!admin|api|_next/static|_next/image|favicon).*)']` to exclude admin, API, and static routes
   - Use NextResponse.next() and set the header on it
  </action>

  </verify>
