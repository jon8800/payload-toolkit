  <files>src/fields/styleOptions.ts, src/fields/link.ts</files>
  <action>
Replace the skeleton styleOptions.ts with the complete style field system. Export a `styleFields` array (not a GroupField -- the array gets spread into the Styles tab fields directly since tabs are unnamed).

The 8 style properties with their field structures:

1. **padding** (group): 4 directional sub-groups (top, right, bottom, left). Each direction has: `base` select (spacingPresets: none/xs/sm/md/lg/xl/2xl/custom), `custom` number (shown via condition when base=custom), and responsive override selects (sm, md, lg, xl) each using spacingPresets.

2. **margin** (group): Same structure as padding.

3. **borderRadius** (group): Single `base` select (none/sm/md/lg/xl/full/custom), `custom` number (condition: base=custom), responsive overrides (sm, md, lg, xl).

4. **borderWidth** (group): Single `base` select (none/1/2/4/8/custom), `custom` number, responsive overrides.

5. **textSize** (group): Single `base` select (xs/sm/base/lg/xl/2xl/3xl/4xl/5xl/custom), `custom` text (for arbitrary Tailwind value like `[18px]`), responsive overrides.

6. **backgroundColor** (group): `preset` select (none/primary/secondary/muted/accent/destructive/background/foreground/card/popover), `custom` text (shown when preset=custom, for hex/rgb values), responsive overrides only for preset (not custom).

7. **textColor** (group): Same structure as backgroundColor with same palette presets.

8. **customCSS** (group): `classes` text field (extra Tailwind classes), `inlineCSS` text field (raw inline CSS string).

Use helper functions to avoid repetition:
- `spacingGroup(name, label)` -- creates the 4-direction spacing group
- `spacingDirection(name, label)` -- creates one direction with base + custom + responsive
- `singlePresetGroup(name, label, presets)` -- creates single-value group with responsive
- `colorGroup(name, label)` -- creates color group with preset + custom

All groups should use `admin: { hideGutter: true }` where appropriate to keep the Styles tab compact. Use Payload `collapsible` groups to keep the tab manageable (per Pitfall 3 from research).

Export: `styleFields` (Field[]), `spacingPresets` (for reference).

For link.ts: Create a reusable link field group with fields:
- `type` select: internal/external (default: internal)
- `url` text (condition: type=external)
- `reference` relationship to pages collection (condition: type=internal) -- use text field with TODO comment since Pages collection is Phase 3
- `label` text (required)
- `newTab` checkbox (default: false)

Export: `linkFields` (Field[]).

<task type="auto">
  <name>Task 2: Build style-to-Tailwind mapper and safelist</name>
  <files>src/lib/blockStyles.ts, src/lib/tailwind-safelist.ts, src/app/(frontend)/globals.css</files>
  <action>
Create src/lib/blockStyles.ts with a `getBlockStyles` function that takes block style data and returns `{ className: string; style?: React.CSSProperties }`.

Mapping logic:
- **Spacing (padding/margin):** Map preset values to Tailwind spacing scale. Presets map: none->0, xs->1, sm->2, md->4, lg->6, xl->8, 2xl->12. For each direction, generate class like `pt-4`, `mr-2`. For responsive, prefix with breakpoint: `sm:pt-4`, `lg:mr-8`. Custom values use arbitrary syntax: `pt-[16px]` (but prefer the Tailwind number).
- **borderRadius:** Map: none->'none', sm->'sm', md->'md', lg->'lg', xl->'xl', full->'full'. Generate `rounded-{value}`, responsive: `sm:rounded-lg`.
- **borderWidth:** Map: none->0, then numeric 1/2/4/8. Generate `border-{value}` or `border` (for 1). Responsive prefixed.
- **textSize:** Map to Tailwind text-size classes: xs->'xs', sm->'sm', base->'base', etc. Generate `text-{value}`. Custom uses arbitrary: `text-[18px]`.
- **backgroundColor:** Preset maps to Tailwind: primary->'bg-primary', secondary->'bg-secondary', etc. Custom hex generates inline style instead.
- **textColor:** Same as backgroundColor but with `text-` prefix.
- **customCSS:** `classes` string appended directly to className. `inlineCSS` parsed into React CSSProperties object (split on `;`, then split each on `:`).

Helper: `parseInlineCSS(css: string): React.CSSProperties` -- converts "max-width: 600px; opacity: 0.5" to `{ maxWidth: '600px', opacity: '0.5' }` (camelCase conversion).

Create src/lib/tailwind-safelist.ts that explicitly lists all possible generated class strings as literal strings in arrays. This ensures Tailwind v4 JIT scanner finds them. Structure:
```typescript
// This file exists solely for Tailwind's JIT scanner.
// The @source directive in globals.css points here.
const spacingClasses = [
  'pt-0', 'pt-1', 'pt-2', 'pt-4', 'pt-6', 'pt-8', 'pt-12',
  'sm:pt-0', 'sm:pt-1', ... // all breakpoint combinations
  // same for pr, pb, pl, mt, mr, mb, ml
]
// ... borderRadius, borderWidth, textSize, bg-*, text-* classes
export default [...spacingClasses, ...borderClasses, ...textClasses, ...colorClasses]
```

Update src/app/(frontend)/globals.css: Add `@source "../lib/tailwind-safelist.ts";` near the top (after the existing @import lines) so Tailwind v4 scans it for class names.
