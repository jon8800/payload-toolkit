  <action>
Create src/components/admin/BlockRowLabel.tsx as a 'use client' component. Adapt the pattern from the luxe reference implementation but support the 14 atomic block types instead of the luxe-specific blocks.

Structure:
1. **Media thumbnail hook** (`useMediaThumbnail`): Same pattern as luxe -- cache media URLs in a Map, fetch from `/api/media/{id}?depth=0`, return thumbnailURL or url. Accept `number | string | null`.

2. **SVG icons** (16x16): Create a unique icon for each of the 14 block types. Use the `Icon` wrapper component pattern from luxe (svg with width=16, height=16, viewBox="0 0 16 16", stroke=currentColor, strokeWidth=1.3). Icons should be semantic:
   - heading: H with horizontal bar (reuse luxe's heading icon)
   - paragraph: text lines (reuse luxe's richText icon)
   - list: bullet points with lines (reuse luxe's bulletList icon)
   - blockquote: quotation mark or indented block
   - image: landscape with sun/mountain (reuse luxe's image icon)
   - video: play button triangle in rectangle
   - icon: star or diamond shape
   - button: rounded rect with text line (reuse luxe's button icon)
   - link: chain link symbol
   - formEmbed: form/clipboard icon (reuse luxe's form icon)
   - container: rectangle/box
   - grid: 2x2 grid squares
   - spacer: vertical arrows pointing apart
   - divider: horizontal line

3. **Block type labels**: Map slug to display name (heading -> 'Heading', paragraph -> 'Paragraph', etc.)

4. **Summary extraction switch**: For each block type, extract:
   - **heading**: `data.text` as summary
   - **paragraph**: extract plain text from `data.content` (Lexical rich text) using `extractPlainText` helper
   - **list**: count of `data.items` array -> "N items"
   - **blockquote**: extract plain text from `data.content`
   - **image**: `mediaId` from `fields[path.image]`, caption as summary
   - **video**: `mediaId` from `fields[path.video]` or `fields[path.poster]`, url as summary for external
   - **icon**: `mediaId` from `fields[path.icon]`, `data.name` as summary
   - **button**: `data.label` as summary (from link fields)
   - **link**: `data.label` as summary
   - **formEmbed**: `data.formTitle` as summary
   - **container**: child count from `data.children` array -> "N children", also search children for first image
   - **grid**: child count + columns setting -> "N items, {columns} cols"
   - **spacer**: just icon + "Spacer" + height setting
   - **divider**: just icon + "Divider"

5. **findFirstImageId**: Walk children blocks (via `fields` using path prefix pattern) looking for image block types with media IDs. Same pattern as luxe.

6. **Render**: Same layout as luxe -- inline-flex span with thumbnail (if mediaId found) or icon, type label, and truncated summary.

Use the `extractPlainText` and `truncate` helpers from luxe (they handle Lexical rich text JSON structure).

The `getNumericValue` helper should handle both number and string IDs (Payload can return either depending on config).

IMPORTANT: The file path in block configs is `'/components/admin/BlockRowLabel.tsx#BlockRowLabel'` -- the export must be a named export called `BlockRowLabel`.
