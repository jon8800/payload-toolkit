  <action>
Redesign all three components to use inline styles with Payload CSS variables instead of Tailwind utility classes. Each component gets a companion SCSS import OR uses inline styles with CSS variables (inline styles preferred to avoid extra SCSS files — keep it simple).

**ColorPickerField redesign:**
- Remove ALL Tailwind classes (mb-6, flex, items-center, gap-2, size-10, rounded, border, etc.)
- Replace with inline styles using CSS variables:
  - Wrapper: `marginBottom: 'calc(var(--base) * 1.5)'`
  - Label: `display: 'block', marginBottom: 'calc(var(--base) * 0.5)', fontSize: 'var(--font-body-size-s)', fontWeight: 500, color: 'var(--theme-text)'`
  - Input row: `display: 'flex', alignItems: 'center', gap: 'calc(var(--base) * 0.5)'`
  - Color input: `width: 40, height: 40, cursor: 'pointer', borderRadius: 'var(--border-radius-m)', border: '1px solid var(--theme-elevation-200)', padding: 2, background: 'var(--theme-input-bg)'`
  - Text input: `borderRadius: 'var(--border-radius-m)', border: '1px solid var(--theme-elevation-200)', padding: 'calc(var(--base) * 0.25) calc(var(--base) * 0.5)', fontSize: 'var(--font-body-size-s)', background: 'var(--theme-input-bg)', color: 'var(--theme-text)'`
  - Preview swatch: `width: 32, height: 32, borderRadius: 'var(--border-radius-m)', border: '1px solid var(--theme-elevation-200)'`
  - Description: `marginTop: 'calc(var(--base) * 0.25)', fontSize: 'var(--font-body-size-s)', color: 'var(--theme-elevation-500)'`

**SliderField redesign:**
- Remove ALL Tailwind classes
- Replace with inline styles using CSS variables:
  - Same wrapper/label/description pattern as ColorPicker
  - Slider row: flex layout with gap
  - Range input: `flex: 1` (native range input needs accent-color set via `accentColor: 'var(--theme-success-500)'`)
  - Number input: styled like Payload native inputs with CSS variables
  - Unit span: `fontSize: 'var(--font-body-size-s)', color: 'var(--theme-elevation-500)'`

**FontSelectorField redesign:**
- Remove ALL Tailwind classes
- Replace with inline styles using CSS variables:
  - Same wrapper/label pattern
  - Selected font display: `fontSize: 'var(--font-body-size-s)', fontWeight: 500, color: 'var(--theme-text)'`
  - Clear button: styled like a subtle Payload button — `background: 'none', border: 'none', cursor: 'pointer', color: 'var(--theme-elevation-500)', fontSize: 'var(--font-body-size-s)'` with hover handled via onMouseEnter/Leave state or just rely on opacity
  - Search input: full Payload-native styling with CSS variables
  - Dropdown: `position: 'absolute', top: '100%', left: 0, zIndex: 50, marginTop: 4, maxHeight: 240, width: '100%', overflowY: 'auto', borderRadius: 'var(--border-radius-m)', border: '1px solid var(--theme-elevation-200)', background: 'var(--theme-elevation-0)', boxShadow: '0 4px 12px rgba(0,0,0,0.1)'`
  - Dropdown items: hover state via `var(--theme-elevation-50)` background
  - Empty state: `color: 'var(--theme-elevation-400)'`

**All components must:**
- Preserve their export names and type signatures exactly
- Preserve all existing functionality (the data flow is unchanged)
- Use NO Tailwind classes — only inline styles with CSS variables or plain CSS properties
- Work in both light and dark admin themes automatically via CSS variables
