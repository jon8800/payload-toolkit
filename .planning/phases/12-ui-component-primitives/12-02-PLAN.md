**Component structure:**
1. Label row (same as current)
2. Inline row with:
   - Color swatch (24x24px rounded square showing current color) — this is the Popover trigger
   - Hex text input (e.g. "#3b82f6") — editable, validates on blur
3. Popover panel (opens on swatch click):
   - Hue slider: a horizontal bar showing the full hue spectrum (0-360), user drags to select hue
   - Saturation/lightness is NOT needed (keep it simple per KISS — just hue + hex input for precise control)
   - Alternatively: render a grid of 18-24 preset color swatches (common colors) that user can click to select
   - Below presets: the hex input is the precise control (already visible in the inline row)
   - A "Clear" button to reset to empty string

**Popover implementation:**
```tsx
<Popover.Root>
  <Popover.Trigger className="color-swatch-trigger" style={{ backgroundColor: value || '#000000' }} />
  <Popover.Portal>
    <Popover.Positioner sideOffset={4}>
      <Popover.Popup className="color-popover">
        {/* Preset color grid */}
        <div className="color-presets">
          {PRESET_COLORS.map(color => (
            <button
              key={color}
              className="color-preset-swatch"
              style={{ backgroundColor: color }}
              onClick={() => setValue(color)}
            />
          ))}
        </div>
        <button className="color-clear-btn" onClick={() => setValue('')}>Clear</button>
      </Popover.Popup>
    </Popover.Positioner>
  </Popover.Portal>
</Popover.Root>
```

Rewrite SliderField.tsx to use Base UI Slider instead of native `<input type="range">`:

**Imports:**
```typescript
'use client'
import type { TextFieldClientComponent } from 'payload'
import { useField } from '@payloadcms/ui'
import { Slider } from '@base-ui/react/slider'
import './SliderField.scss'
```

**Keep unchanged:**
- Export: `export const SliderField: TextFieldClientComponent = function SliderField({ path, field })`
- useField<string>({ path }) for reading/writing
- Reading custom props: field.admin?.custom.{min, max, step, unit}
- Label and description rendering
- Number input for precise value entry

**Component structure (Framer-style: label + slider + value on one row per D-25, D-35):**
```tsx
<div className="slider-field">
  <label className="slider-field__label">{label}</label>
  <div className="slider-field__controls">
    <Slider.Root
      value={numericValue}
      onValueChange={(val) => handleChange(val)}
      min={min}
      max={max}
      step={step}
      className="slider-field__slider"
    >
      <Slider.Control className="slider-field__control">
        <Slider.Track className="slider-field__track">
          <Slider.Indicator className="slider-field__indicator" />
          <Slider.Thumb className="slider-field__thumb" />
        </Slider.Track>
      </Slider.Control>
    </Slider.Root>
    <input
      type="number"
      className="slider-field__number"
      min={min} max={max} step={step}
      value={numericValue}
      onChange={(e) => handleChange(parseFloat(e.target.value))}
    />
    {unit && <span className="slider-field__unit">{unit}</span>}
  </div>
  {description && <p className="slider-field__description">{description}</p>}
</div>
```

**Note on Base UI Slider v1.3.0 API:** The `onValueChange` callback receives the new value directly (a number for single-value sliders). Check the actual API — it may be `onValueChange(value: number, event)` or `onValueChange(value: number[])`. Read the Base UI Slider docs if needed. The value prop also accepts a number directly for single-thumb sliders.

**handleChange function:**
```typescript
function handleChange(newValue: number | number[]) {
  const val = Array.isArray(newValue) ? newValue[0] : newValue
  setValue(String(val))
}
```
