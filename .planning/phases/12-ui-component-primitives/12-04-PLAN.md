**Simpler approach — add admin.style to the ThemeSettings config:**
Payload supports `admin.style` on the global config for custom CSS. But this only adds to the wrapper. The cleanest solution is:

1. In `index.scss`, add a rule for the default global edit view that limits field width:
```scss
// Theme settings compact layout
[data-global-slug="theme-settings"] {
  .render-fields {
    max-width: 480px;
  }

  // Compact group spacing
  .group-field {
    margin-bottom: calc(var(--base) * 0.5);
  }

  // Group labels as subtle section headers
  .group-field__wrap > label {
    font-size: 13px;
    font-weight: 600;
    color: var(--theme-text);
    text-transform: uppercase;
    letter-spacing: 0.04em;
    padding-bottom: calc(var(--base) * 0.25);
    border-bottom: 1px solid var(--theme-elevation-100);
    margin-bottom: calc(var(--base) * 0.5);
  }
}
```

Note: The exact Payload DOM selectors may differ. The executor should inspect the rendered HTML to find the correct selectors. The `[data-global-slug="theme-settings"]` attribute is a common Payload pattern for targeting specific global edit views — if it does not exist, use `.global-edit--theme-settings` or similar.
