In ThemeSettings.ts, add `livePreview` to the `admin` config. For globals, the URL function receives `{ req }` (no `data.slug` like collections).

The style guide page is at `/style-guide`. Use the same preview path pattern as collections but with a static path.

Update the `admin` block:

```ts
admin: {
  group: 'Settings',
  livePreview: {
    url: ({ req }) => {
      const params = new URLSearchParams({
        slug: 'style-guide',
        collection: '',
        path: '/style-guide',
        previewSecret: process.env.PREVIEW_SECRET || '',
      })
      return `/next/preview?${params.toString()}`
    },
  },
},
```

This follows the same pattern as `generatePreviewPath` but hardcoded to the style guide route. The `/next/preview` endpoint is Payload's built-in preview route that sets draft mode and redirects to the target path.

IMPORTANT: The `generatePreviewPath` utility expects a `collection` param, but for a global config this pattern is simpler. Do NOT import generatePreviewPath — write inline since the global has no slug/collection.
