1. Modify `apps/starter/src/app/(frontend)/layout.tsx`:

   a. Add imports at top:
      ```typescript
      import { unstable_cache } from 'next/cache'
      import { getPayload } from 'payload'
      import configPromise from '@payload-config'
      import { buildCSSVariables } from '@/lib/themeUtils'
      ```

   b. Create cached theme fetcher OUTSIDE the component (module scope):
      ```typescript
      const getTheme = unstable_cache(
        async () => {
          const payload = await getPayload({ config: configPromise })
          return payload.findGlobal({ slug: 'theme-settings' })
        },
        ['theme-settings'],
        { tags: ['theme-settings'] }
      )
      ```

   c. Inside `RootLayout`, fetch theme at the top (after existing draftMode/headers calls):
      ```typescript
      const theme = await getTheme()
      const cssVars = buildCSSVariables(theme as any)
      ```
      Use `as any` or cast to ThemeData since Payload's generated type may not match exactly. buildCSSVariables already handles null/undefined fields gracefully.

   d. Apply CSS variables as inline style on `<html>`:
      ```typescript
      <html
        className={cn(GeistSans.variable, GeistMono.variable)}
        lang="en"
        style={Object.keys(cssVars).length > 0 ? cssVars : undefined}
      >
      ```
      Only set style if there are actual overrides (zero-config = no inline style = globals.css defaults apply).

   e. Add Google Fonts link tags in `<head>`, BEFORE the favicon links:
      ```typescript
      {theme?.fonts?.sans && (
        <>
          <link rel="preconnect" href="https://fonts.googleapis.com" />
          <link rel="preconnect" href="https://fonts.gstatic.com" crossOrigin="anonymous" />
          <link
            rel="stylesheet"
            href={`https://fonts.googleapis.com/css2?family=${encodeURIComponent(theme.fonts.sans)}&display=swap`}
          />
        </>
      )}
      {theme?.fonts?.mono && (
        <link
          rel="stylesheet"
          href={`https://fonts.googleapis.com/css2?family=${encodeURIComponent(theme.fonts.mono)}&display=swap`}
        />
      )}
      ```
      Note: preconnect tags only need to appear once, so put them with the sans font block (which is more commonly set).

   f. Do NOT remove the existing GeistSans/GeistMono className — Geist remains the static fallback. The theme's `--font-sans` CSS variable (set by buildCSSVariables) will override `--font-geist-sans` only when a custom font is configured.

2. Keep all existing layout functionality intact — template parts resolution, draftMode, BlockSelectionHandler, header/footer rendering. Only ADD the theme fetch and injection.

Important: The `buildCSSVariables` function from themeUtils.ts should output variables like `--primary`, `--background`, `--radius`, `--font-sans`, etc. These directly override the :root values from globals.css. The @theme inline block in globals.css then maps these to Tailwind utility names (--color-primary = var(--primary)), so Tailwind classes like `bg-primary` automatically pick up the new values.
