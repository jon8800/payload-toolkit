  <name>Task 1: Create FormClient client component</name>
  <files>apps/starter/src/blocks/FormEmbed/FormClient.tsx</files>
  <action>
Create a 'use client' component at `apps/starter/src/blocks/FormEmbed/FormClient.tsx` that receives the form definition object and renders interactive form fields.

Props interface:
```typescript
type FormClientProps = {
  formId: string
  fields: Array<{
    blockType: string
    name?: string
    label?: string
    required?: boolean
    defaultValue?: string | boolean
    width?: number
    options?: Array<{ label: string; value: string }>
    message?: object
  }>
  submitButtonLabel?: string
  confirmationType?: 'message' | 'redirect'
  confirmationMessage?: string
  redirectUrl?: string
}
```

Implementation details:
- Use useState for form data (Record<string, string | boolean>) and submission state (idle | submitting | success | error)
- Render each field based on blockType using a switch/map:
  - 'text': `<input type="text">` with label, name, required, defaultValue
  - 'email': `<input type="email">` with label, name, required
  - 'textarea': `<textarea>` with label, name, required
  - 'select': `<select>` with `<option>` elements from field.options
  - 'checkbox': `<input type="checkbox">` with label
  - 'number': `<input type="number">`
  - 'message': render as static `<p>` (informational, not a form input). Use dangerouslySetInnerHTML only if message is a string, otherwise skip rendering complex rich text objects for now.
  - 'country', 'state': `<select>` (render as text input as basic fallback -- full country/state lists are out of scope for v1 gap closure)
- Create a buildFieldRenderer helper function that returns the appropriate JSX for each field type to keep the component DRY
- Wrap each field in a `<div>` with `<label>` above the input. Apply basic Tailwind classes: `flex flex-col gap-1.5 mb-4` on wrapper, `text-sm font-medium` on label, standard input styling via `border rounded-md px-3 py-2 text-sm` on inputs
- If field.width is set (1-100), apply it as `style={{ width: '${field.width}%' }}` on the wrapper div
- On submit: prevent default, set state to submitting, POST to `${process.env.NEXT_PUBLIC_SERVER_URL || ''}/api/form-submissions` with body `{ form: formId, submissionData }` where submissionData maps form data to `[{ field: name, value: stringValue }]`
- On success (2xx): set state to 'success', show confirmationMessage text or "Form submitted successfully" as default
- On error: set state to 'error', show "Something went wrong. Please try again."
- Submit button: use `<button type="submit">` with submitButtonLabel or "Submit" as fallback. Disable while submitting.
- If confirmationType is 'redirect' and redirectUrl exists, use `window.location.href = redirectUrl` after successful submission instead of showing confirmation message
- Use early returns: if state is 'success' and not redirect, render the success message and return early before the form

Use standard function declarations (not arrow functions) for the component per project conventions.
