**2. Create `apps/starter/src/migrations/20260315_styles_json.ts`:**

The migration must:

A. **Define the style field names to extract:**
```typescript
const STYLE_FIELDS = ['padding', 'margin', 'borderRadius', 'borderWidth', 'textSize', 'backgroundColor', 'textColor', 'customCSS']
```

B. **Create a recursive helper `migrateBlocks(blocks)`:**
- Iterate over each block in the array
- For each block, check if any of the STYLE_FIELDS exist at the root level (and `styles` field does NOT already exist — idempotency)
- If old fields exist: create a `styles` object, move each STYLE_FIELD value into it, then delete the root-level field
- Also check for `customClasses` and `inlineCSS` at root (from old settingsTab) — if present, merge them into `styles.customCSS`:
  - Append `customClasses` to `styles.customCSS.classes` (space-separated, avoid duplicates)
  - Append `inlineCSS` to `styles.customCSS.inlineCSS` (semicolon-separated)
  - Delete root-level `customClasses` and `inlineCSS`
- **Recurse into nested blocks:** Check for `children` array (Container/Grid blocks have nested children). Apply the same migration to each nested block.
- Return whether any changes were made (boolean)

C. **`up()` function:**
- Use `payload.find()` on the `pages` collection with `limit: 0` (get all), `depth: 0`
- For each page, run `migrateBlocks(page.layout)` on the layout blocks array
- If changes were made, use `payload.update()` to save the page
- Log progress: `Migrated page: {page.title}`

D. **`down()` function:**
- Reverse: for each page, iterate blocks, if `styles` field exists, spread its properties back to root level, delete `styles`
- Handle customCSS reversal: split `styles.customCSS.classes` and `styles.customCSS.inlineCSS` back to root-level `customClasses`/`inlineCSS` (best effort — just copy them)
- Recurse into children

**3. Update `apps/starter/src/migrations/index.ts`:**
Add the new migration to the exports array, following the existing pattern.

**Important:** The migration should be idempotent — if run twice, it should not corrupt data. Check for `styles` field existence before migrating.
