# Requirements: Payload CMS Starter

**Defined:** 2026-03-15
**Core Value:** Any website project can be scaffolded instantly with a composable block-based layout system that works for both static pages and dynamic collection templates

## v1.1 Requirements

### Frontend Styling

- [x] **STYLE-01**: Frontend pages render with Tailwind v4 Preflight reset (proper fonts, heading sizes, base styles) — *v1.1*
- [x] **STYLE-02**: Block components apply configured style properties (padding, margin, colors, etc.) visually on frontend — *v1.1*
- [x] **STYLE-03**: shadcn/ui components render correctly on frontend pages (buttons, forms, cards) — *v1.1*

### Styles Panel

- [x] **PANEL-01**: Per-block styles stored as single JSON field instead of individual Payload fields — *v1.1*
- [x] **PANEL-02**: Custom admin component with Webflow-style bounding box UI for margin and padding (4-side editing) — *v1.1*
- [x] **PANEL-03**: Collapsible property groups for border, typography, colors, and custom CSS — *v1.1*
- [x] **PANEL-04**: Style changes persist and render correctly on frontend — *v1.1*
- [x] **PANEL-05**: Existing block style data migrated to new JSON format — *v1.1*

### Theme Settings

- [x] **THEME-01**: Theme Settings global with JSON field storing site-wide design tokens (colors, fonts, spacing, border radius) — *v1.1*
- [x] **THEME-02**: Custom admin view for editing theme values visually — *v1.1*
- [x] **THEME-03**: Theme values injected as CSS variables on frontend pages via layout.tsx — *v1.1*
- [x] **THEME-04**: shadcn/ui components respect theme CSS variables — *v1.1* (livePreview route fix delivered as post-audit commit 23d3250)

### Admin Component UX

- [x] **UI-01**: StylesPanel bounding box redesigned Webflow-style — centered, clean, unified colors, professional property controls — *v1.1*
- [x] **UI-02**: Custom CSS input uses a code editor with syntax highlighting (not a text input) — *v1.1*
- [x] **UI-03**: Tailwind classes input uses tag-based chip tokens (add/remove, not plain text) — *v1.1*
- [x] **UI-04**: Theme field components (ColorPicker, SliderField, FontSelector) redesigned with Payload-native CSS variables aesthetic — *v1.1*
- [x] **UI-05**: Customizer right sidebar excess horizontal padding removed — *v1.1*

## Deferred to Future Milestones

### Templates & Dynamic Content

- **TMPL-01**: Templates collection for dynamic page layouts (Elementor-style)
- **TMPL-02**: Type-safe dynamic data binding (block properties reference compatible collection field types only)

### Search

- **SRCH-01**: Frontend search UI consuming the Search plugin

## Out of Scope (v1.1)

| Feature | Reason |
|---------|--------|
| Full CSS property abstraction (Webflow-level) | Simplified subset — exponential complexity |
| Per-page theme overrides | Global theme only for v1.1; per-page theming deferred |
| CSS-in-JS or styled-components | Tailwind v4 CSS-first approach, CSS variables only |
| Dark mode toggle | Can be added via theme settings later, not shipped in v1.1 |
