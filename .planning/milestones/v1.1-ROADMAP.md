# Milestone v1.1: Styling & Theming

**Status:** ✅ SHIPPED 2026-04-18
**Phases:** 8, 9, 10, 11, 12, 12.1
**Total Plans:** 16 plans

## Overview

Fix frontend Tailwind v4 rendering, replace per-field block styles with a single JSON field and Webflow-inspired visual panel, add a Theme Settings global that cascades design tokens as CSS variables, then redesign the custom admin components twice — first for Payload-native aesthetic, then with Base UI primitives — culminating in a style guide preview page for live theme editing.

## Milestone Summary

**Decimal Phases:**
- Phase 12.1: UI Iteration Fixes (inserted after Phase 12 for visual bugs + livePreview wiring)

**Key Decisions:**
- Auto-compute foreground colors based on oklch lightness; chart colors via primary hue rotation (Phase 10)
- Fixed Tailwind v4 CSS with @source directives — CSS output grew from ~91 to 6839 lines (Phase 8)
- Styles panel replaces per-field CSS properties with single JSON field + custom bounding box UI (Phase 9)
- Theme Settings stores design tokens as JSON, injects as CSS variables via layout.tsx (Phase 10)
- Used static curated list of 50 Google Fonts in Phase 10 then expanded to full 1908-font catalog in Phase 12 (google-font-metadata data file)
- JSONField type for stylesField factory; bounding box UI edits base values only, responsive deferred (Phase 9)
- Payload API (find/update) for data migration instead of raw SQL for cross-adapter compatibility (Phase 9)
- CSS variables applied via `<style>` tag in `<head>` (cascade to `:root`) instead of inline `style` on `<html>` element (Phase 10)
- Admin field components use inline styles with Payload CSS variables, no Tailwind in admin context (Phase 11)
- FontSelector hover effects via useState since inline styles cannot use :hover (Phase 11)
- CodeEditor imported via named export from @payloadcms/ui root (Phase 11)
- ClassTokenInput stores chips as space-separated string for backward compatibility (Phase 11)
- Base UI Popover for ColorPicker and Slider for SliderField, replacing native widgets (Phase 12)
- Base UI Combobox with IntersectionObserver lazy font preview for 1908 Google Fonts (Phase 12)
- Reusable Section component wrapping Base UI Collapsible for DRY (Phase 12)
- Spacing section always visible (not collapsible) as primary control; color swatch as styled span (Phase 12)
- Kept borderRadius at root level to preserve data path compatibility (Phase 12)
- StyledSelect wrapper component inside StylesPanel.tsx for DRY Base UI Select usage (Phase 12.1)
- react-colorful HexColorPicker for color wheel; theme color defaults from shadcn oklch-to-hex values (Phase 12.1)
- Inline livePreview URL for global config instead of generatePreviewPath utility (Phase 12.1)

**Issues Resolved:**
- Frontend Tailwind v4 rendering broken on shipped v1.0 — fixed via @source directives
- Per-field block styles cluttering schema — consolidated into single JSON field with migration
- Theme values hardcoded — externalized to Theme Settings global with CSS variable injection
- Admin components looked like a prototype — redesigned twice (Payload-native then Base UI primitives)
- Native browser selects/color pickers/range sliders inconsistent across browsers — replaced with Base UI primitives
- livePreview 404 for globals — `/next/preview` route updated to make `collection` optional (post-audit fix 23d3250)

**Issues Deferred:**
- Responsive style overrides in StylesPanel UI (schema supports it; UI shows base values only)
- Human verification of Phase 10 admin UI (all programmatic checks pass; visual confirmation pending)
- 12-VERIFICATION.md and 12.1-VERIFICATION.md (phase verification docs never written; integration check confirmed wiring)
- Nyquist validation setup across milestone (5/6 phases missing VALIDATION.md)

**Technical Debt Incurred:**
- CSS variable injection uses `<style>` tag in `<head>` rather than inline `style` attribute on `<html>` — functionally equivalent, docs/implementation alignment only
- `ThemeSettings.livePreview.url` still passes `collection: ''` (harmless after route fix — route no longer requires it)
- No test coverage for any phase (pre-existing repo convention; Nyquist gap)
