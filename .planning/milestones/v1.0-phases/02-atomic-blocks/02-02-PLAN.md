```typescript
// src/blocks/slugs.ts — Single source of truth for block slug arrays.
// Both block configs (for blockReferences) and registry.ts import from here.
// This breaks the circular dependency: configs -> slugs <- registry.
export const atomicBlockSlugs = [
  'heading', 'paragraph', 'list', 'blockquote',
  'image', 'video', 'icon',
  'button', 'link', 'formEmbed',
  'container', 'grid', 'spacer', 'divider',
] as const

export type AtomicBlockSlug = (typeof atomicBlockSlugs)[number]
```

THEN — Create the 4 text block configs. Every block follows this exact pattern:
- `interfaceName: '{Name}Block'` for TypeScript types
- `admin.disableBlockName: true`
- `admin.components.Label: '/components/admin/BlockRowLabel.tsx#BlockRowLabel'`
- Three UNNAMED tabs (label only, no name property — critical to avoid data nesting):
  - Content tab: block-specific fields + children blocks field
  - Styles tab: `fields: styleFields` (imported from `@/fields/styleOptions`)
  - Settings tab: `htmlTag` select + `customClasses` text + `inlineCSS` text + block-specific settings

Children field on EVERY block (including all 14 — no exceptions per user decision):
```typescript
{
  name: 'children',
  type: 'blocks',
  blockReferences: atomicBlockSlugs,  // imported from @/blocks/slugs
  blocks: [],
  maxRows: 20,
  admin: { description: 'Nested child blocks' },
  validate: (value, { siblingData, path }) => {
    // Enforce 8-level max nesting depth
    // path is an array like ['layout', '0', 'children', '0', 'children', '0', ...]
    // Count occurrences of 'children' in the path to determine depth
    const depth = (path || []).filter(segment => segment === 'children').length
    if (depth > 8) {
      return 'Maximum nesting depth of 8 levels exceeded.'
    }
    return true
  },
}
```

NOTE: The validate function checks nesting depth by counting 'children' segments in the field path. Payload passes the full field path, so a block nested 9 levels deep would have 9 'children' segments. If Payload's path structure differs (e.g., dot-separated string), split on '.' first. Test the exact path format and adjust accordingly.

Block-specific Content and Settings fields:

**Heading**: Content: `text` (text, required), `tag` (select: h1-h6, default h2). Settings: htmlTag default 'header'.
**Paragraph**: Content: `content` (richText — uses Lexical editor). Settings: htmlTag default 'div'.
**List**: Content: `items` (array of `{ text: text, required: true }`), `listType` (select: unordered/ordered, default unordered). Settings: htmlTag default 'div'.
**Blockquote**: Content: `content` (richText), `citation` (text). Settings: htmlTag default 'blockquote'.

IMPORTANT: Use the research patterns exactly. Tabs must be unnamed (no `name` property). The Label component path must be `'/components/admin/BlockRowLabel.tsx#BlockRowLabel'` (relative to importMap.baseDir which is src/).
