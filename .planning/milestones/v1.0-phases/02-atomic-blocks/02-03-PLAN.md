Create each block component as a React Server Component (no 'use client'). Use standard function declarations (not arrow functions) per project conventions.

Every component follows this pattern:
1. Import `cn` from `@/lib/utils`, `getBlockStyles` from `@/lib/blockStyles`, `RenderBlocks` from `@/blocks/RenderBlocks`
2. Accept props matching the block's data shape (use inline types since payload-types.ts may not be regenerated yet — define a Props type per component)
3. Extract `htmlTag`, `customClasses`, `inlineCSS`, `children`, and style-related fields
4. Call `getBlockStyles` with the style data to get className and style
5. Render using the configurable `htmlTag` as the wrapper element
6. Merge styles: `cn(className, customClasses)` for classes, spread inline styles
7. Render children via `<RenderBlocks blocks={children} />` if children exist

Text block specifics:
- **Heading**: Render `tag` (h1-h6) inside the wrapper. `<HeadingTag>{text}</HeadingTag>`.
- **Paragraph**: Use the RichText component from `@payloadcms/richtext-lexical/react` to render Lexical content. Import: `import { RichText } from '@payloadcms/richtext-lexical/react'`. Usage: `<RichText data={content} />`. If this import doesn't exist, try `@payloadcms/richtext-lexical` or fall back to rendering a simple div with a TODO comment.
- **List**: Render `<ul>` or `<ol>` based on `listType`, map items to `<li>` elements.
- **Blockquote**: Render `<blockquote>` with rich text content and optional `<cite>` for citation.

IMPORTANT: These are ALL server components. No 'use client' directive. No useState, useEffect, or event handlers.
