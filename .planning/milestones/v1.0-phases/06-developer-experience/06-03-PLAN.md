**Install dependencies first:**
```bash
pnpm add -D @clack/prompts pg @types/pg tsx
```

**scripts/setup.ts** (runs via `tsx scripts/setup.ts`, NOT `payload run`):
- Import `* as p from '@clack/prompts'`
- Import helpers: `createDatabase`, `generateEnv`, `modifyConfig` from `./lib/`
- Import `execSync` from `child_process`
- Import `existsSync` from `fs`

Flow:
1. `p.intro('Payload Starter Setup')`
2. Check if `.env` already exists -- if so, `p.confirm()` whether to overwrite or skip
3. `p.group()` with sequential prompts:
   - `name`: project name (default: 'my-website', validate: lowercase + hyphens + numbers only)
   - `dbHost`: database host (default: 'localhost')
   - `dbPort`: database port (default: '5432')
   - `dbUser`: database user (default: 'postgres')
   - `dbPassword`: database password (required)
   - `adminEmail`: admin email (validate: contains @)
   - `adminPassword`: admin password (validate: min 8 chars)
   - `collections`: `p.multiselect()` for optional collections -- Posts, Categories, Tags, Template Parts. All selected by default via `initialValues`.
   - `plugins`: `p.multiselect()` for optional plugins -- SEO, Redirects, Form Builder, Nested Docs, Import/Export, MCP, Search. All selected by default.
   - `smtpSetup`: `p.confirm()` whether to configure SMTP (default: false)

4. Conditional SMTP group (only if smtpSetup is true):
   - host, port (default 587), user, pass, from

5. Execute steps with `p.spinner()`:
   a. Create database: `await createDatabase({ host, port: Number(port), user, password, database: name })`
   b. Generate .env: `generateEnv({ projectName: name, dbHost, dbPort, dbUser, dbPassword, smtp })`
   c. Modify config: `modifyConfig(collections as string[], plugins as string[])`
   d. Run migrations: `execSync('pnpm payload migrate', { stdio: 'inherit' })`
   e. Create admin user: `execSync('pnpm payload run scripts/create-admin.ts -- --email="..." --password="..."', { stdio: 'inherit' })`

6. `p.outro()` with success message and next steps: "Run `pnpm dev` to start developing"

7. Handle cancellation: `onCancel` in group calls `p.cancel()` and `process.exit(0)`

8. Wrap entire flow in try/catch -- on error, `p.log.error()` and exit 1

**scripts/create-admin.ts** (runs via `payload run`):
- Import `getPayload` from `'payload'` and config from `'@payload-config'`
- Parse `--email` and `--password` from `process.argv`
- `const payload = await getPayload({ config })`
- Check if any users exist: `const existing = await payload.find({ collection: 'users', limit: 1 })`
- If no users exist, create one: `await payload.create({ collection: 'users', data: { email, password } })`
- Log success and `process.exit(0)`

**package.json** -- add to `scripts`:
```json
"setup": "tsx scripts/setup.ts",
"seed:demo": "cross-env NODE_OPTIONS=--no-deprecation payload run ./scripts/seed-demo.ts",
"create:admin": "cross-env NODE_OPTIONS=--no-deprecation payload run ./scripts/create-admin.ts"
```
