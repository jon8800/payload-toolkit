1. Create `.env.example` with all env vars documented:
   - DATABASE_URL (with format example)
   - PAYLOAD_SECRET

   - PREVIEW_SECRET
   - SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS, SMTP_FROM, SMTP_SECURE (commented out with descriptions, note "Works with any SMTP provider")

2. Add `@setup:` marker comments to `src/payload.config.ts`. Each optional item gets a marker on BOTH its import line AND its usage:

   **Collections** (Users, Media, Pages are `@setup:required`; the rest are optional):
   - `import { Posts }` line: append `// @setup:collection:posts`
   - `import { Categories }` line: append `// @setup:collection:categories`
   - `import { Tags }` line: append `// @setup:collection:tags`
   - `import { TemplateParts }` line: append `// @setup:collection:template-parts`
   - In the `collections: [...]` array, same markers on each optional entry
   - Note: Categories depends on nestedDocsPlugin -- the modify-config script handles this coupling

   **Plugins** (use start/end markers for multi-line configs):
   - nestedDocsPlugin: `// @setup:plugin:nested-docs:start` before, `// @setup:plugin:nested-docs:end` after
   - seoPlugin: `// @setup:plugin:seo:start` / `// @setup:plugin:seo:end`
   - redirectsPlugin: `// @setup:plugin:redirects:start` / `// @setup:plugin:redirects:end`
   - formBuilderPlugin: `// @setup:plugin:form-builder:start` / `// @setup:plugin:form-builder:end`
   - importExportPlugin: `// @setup:plugin:import-export:start` / `// @setup:plugin:import-export:end`
   - mcpPlugin: `// @setup:plugin:mcp:start` / `// @setup:plugin:mcp:end`
   - searchPlugin: `// @setup:plugin:search:start` / `// @setup:plugin:search:end`
   - Plugin imports also get single-line markers: `// @setup:plugin:seo` etc.

3. Add SMTP email transport to payload.config.ts:
   - Import nodemailer at the top (already a project dependency)
   - Add `email` property to the buildConfig object:
     ```typescript
     email: nodemailer.createTransport({
       host: process.env.SMTP_HOST,
       port: Number(process.env.SMTP_PORT) || 587,
       secure: process.env.SMTP_SECURE === 'true',
       auth: process.env.SMTP_USER
         ? { user: process.env.SMTP_USER, pass: process.env.SMTP_PASS }
         : undefined,
     }),
     ```
   - Add `fromAddress` and `fromName` to buildConfig:
     ```typescript
     email: {
       transportOptions: {
         host: process.env.SMTP_HOST,
         port: Number(process.env.SMTP_PORT) || 587,
         secure: process.env.SMTP_SECURE === 'true',
         auth: process.env.SMTP_USER
           ? { user: process.env.SMTP_USER, pass: process.env.SMTP_PASS }
           : undefined,
       },
       fromAddress: process.env.SMTP_FROM || 'noreply@example.com',
       fromName: 'Payload Starter',
     },
     ```
   - Check Payload 3.x docs pattern for email config -- it may use `email` adapter directly. Use the `nodemailerAdapter` from `@payloadcms/email-nodemailer` if available, otherwise use the transport options pattern above. If the adapter package doesn't exist at Payload 3.79, use direct transportOptions.

IMPORTANT: Do NOT add the `email` config if it would break the build when SMTP vars are absent. The transport should gracefully handle missing SMTP env vars (already does via the conditional auth and fallback values). Wrap in a conditional if needed: only add email config when SMTP_HOST is defined, or always add with safe fallbacks.

<task type="auto">
  <name>Task 2: Setup helper scripts (create-database, generate-env, modify-config, lexical-helpers)</name>

  <action>
Create 4 helper modules in `scripts/lib/`. These are plain TypeScript files run by tsx (NOT payload run), so they should NOT import Payload or use @payload-config alias. Use standard Node.js imports only.

**scripts/lib/create-database.ts:**
- Export `createDatabase(config: { host: string; port: number; user: string; password: string; database: string }): Promise<void>`
- Import `Client` from `pg`
- Connect to the default `postgres` database first (NOT the target database)
- Check if target DB exists: `SELECT 1 FROM pg_database WHERE datname = $1`
- If not, create it: `CREATE DATABASE "${dbName}"` (quote for hyphens)
- Always `client.end()` in a finally block
- Log success/skip message to console

**scripts/lib/generate-env.ts:**
- Export `generateEnv(options: { projectName: string; dbHost: string; dbPort: string; dbUser: string; dbPassword: string; adminEmail?: string; smtp?: { host: string; port: string; user: string; pass: string; from: string } }): void`
- Read `.env.example` as template
- Replace placeholder values with actual user inputs:
  - DATABASE_URL: construct from parts `postgresql://${user}:${password}@${host}:${port}/${projectName}`
  - PAYLOAD_SECRET: generate a random 32-char hex string via `crypto.randomBytes(16).toString('hex')`
  - PREVIEW_SECRET: generate another random string
  - NEXT_PUBLIC_SERVER_URL: keep as http://localhost:3000
  - SMTP vars: uncomment and fill if smtp config provided, leave commented if not
- Write result to `.env`
- Do NOT overwrite if `.env` already exists -- warn and skip, or prompt (caller decides)

**scripts/lib/modify-config.ts:**
- Export `modifyConfig(selectedCollections: string[], selectedPlugins: string[]): void`
- Read `src/payload.config.ts` as text
- For each collection NOT in `selectedCollections`, find lines with `// @setup:collection:{name}` and prepend `// ` to comment them out
- For each plugin NOT in `selectedPlugins`:
  - Comment out import lines with `// @setup:plugin:{name}`
  - Comment out all lines between `// @setup:plugin:{name}:start` and `// @setup:plugin:{name}:end` (inclusive)
- Handle dependency coupling: if 'categories' is deselected, also deselect 'nested-docs' plugin. If 'posts' is deselected, also deselect 'categories' and 'tags'.
- Write the modified file back
- Use simple line-by-line string processing (NOT regex soup or AST manipulation)

**scripts/lib/lexical-helpers.ts:**
- Export `richText(...paragraphs: string[])` -- returns valid Lexical JSON root with paragraph nodes
- Export `richTextWithHeading(heading: string, ...paragraphs: string[])` -- heading node + paragraph nodes
- Export `populatePresetContent(blocks: Record<string, unknown>[], contentMap: Record<string, string>): Record<string, unknown>[]`
  - Deep-clones blocks, finds `blockType: 'paragraph'` with `content: null`, fills with richText using contentMap lookup or fallback lorem ipsum
  - Recursively processes `children` arrays for nested blocks (containers, grids)
- Use the exact Lexical JSON structure from the research:
  ```
  { root: { type: 'root', children: [...], direction: 'ltr', format: '', indent: 0, version: 1 } }
  ```
  Each paragraph: `{ type: 'paragraph', children: [{ type: 'text', detail: 0, format: 0, mode: 'normal', style: '', text, version: 1 }], direction: 'ltr', format: '', indent: 0, textFormat: 0, textStyle: '', version: 1 }`
