<objective>
Fix Tailwind v4 content scanning so all utility classes from block components, UI components, and layout files are included in the generated CSS.

Purpose: Frontend pages currently render as unstyled HTML because Tailwind v4's automatic content scanner only looks in the CSS file's directory by default. The app's components live in `src/blocks/`, `src/components/`, `src/lib/`, etc., which are outside the scan path. Adding `@source` directives tells Tailwind v4 exactly where to find files containing utility classes.

Output: Working frontend CSS with Preflight reset, block styles, and shadcn/ui styling
</objective>

  <name>Task 1: Add Tailwind v4 source directives for all component directories</name>
  <files>apps/starter/src/app/(frontend)/globals.css</files>
  <action>
Add @source directives to globals.css so Tailwind v4 scans all directories containing files with Tailwind utility classes. Add these directives AFTER the existing `@source "../../lib/tailwind-safelist.ts";` line (which must be kept for dynamic class safelisting):

```css
@source "../../blocks/**/*.tsx";
@source "../../components/**/*.tsx";
@source "../**/*.tsx";
@source "../../lib/**/*.ts";
```

These four directives cover:
- `../../blocks/**/*.tsx` — all block component files (Heading, Container, Grid, Button, etc.)
- `../../components/**/*.tsx` — all shadcn/ui components and custom components (BlockSelectionHandler, etc.)
- `../**/*.tsx` — layout.tsx, page.tsx, and other app directory files (relative to globals.css in `(frontend)/`)
- `../../lib/**/*.ts` — utility files like blockStyles.ts, utils.ts

Keep the existing `@source "../../lib/tailwind-safelist.ts";` line as-is — it handles dynamically composed classes from blockStyles that cannot be detected by static scanning.

Do NOT modify any other part of globals.css. Only add the @source lines.
