The globals.css `:root` block defines these shadcn color variables in oklch format. The hex equivalents are:

| CSS Variable | oklch value | Hex equivalent |
|---|---|---|
| --primary | oklch(0.205 0 0) | #0a0a0a |
| --secondary | oklch(0.97 0 0) | #f5f5f5 |
| --accent | oklch(0.97 0 0) | #f5f5f5 |
| --muted | oklch(0.97 0 0) | #f5f5f5 |
| --destructive | oklch(0.577 0.245 27.325) | #ef4444 |
| --background | oklch(1 0 0) | #ffffff |
| --foreground | oklch(0.145 0 0) | #171717 |

Add `defaultValue` to each color field in ThemeSettings.ts:

```ts
{
  name: 'primary',
  type: 'text',
  defaultValue: '#0a0a0a',
  admin: {
    description: 'Default: #0a0a0a (oklch(0.205 0 0))',
    components: {
      Field: '@/fields/theme/ColorPicker#ColorPickerField',
    },
  },
},
{
  name: 'secondary',
  type: 'text',
  defaultValue: '#f5f5f5',
  admin: {

    // ...
  },
},
// ... same pattern for accent (#f5f5f5), muted (#f5f5f5), destructive (#ef4444), background (#ffffff), foreground (#171717)
```

This ensures new ThemeSettings documents start with the correct shadcn color values instead of empty/black.
