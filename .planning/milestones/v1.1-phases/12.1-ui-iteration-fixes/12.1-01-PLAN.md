  <action>
In FontSelector.scss, the `.font-selector__popup` class currently has no explicit width, so the Base UI Combobox positions it at its natural content width.

Add width matching to the popup by setting the Positioner to match trigger width. Base UI Combobox.Positioner supports a `style` prop, but the CSS approach is cleaner.

Add to `.font-selector__popup`:
```scss
.font-selector__popup {
  // existing styles...
  width: var(--anchor-width, 100%);  // Base UI sets --anchor-width CSS variable on positioner
}
```

If `--anchor-width` is not automatically provided by Base UI's Combobox.Positioner, then instead modify FontSelector.tsx to add `trackAnchorWidth` prop to the Combobox.Positioner:
```tsx
<Combobox.Positioner sideOffset={4} trackAnchorWidth>
```
This tells Base UI to set the popup width to match the trigger width.

Also ensure the `.font-selector__popup` has `min-width: 200px` as a floor so it never looks too narrow.

IMPORTANT: Do NOT use Tailwind classes. SCSS + Payload CSS variables only.
