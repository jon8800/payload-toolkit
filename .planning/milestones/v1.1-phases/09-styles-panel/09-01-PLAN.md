```typescript
// Data shape the JSON field stores (matches what getBlockStyles expects)
type SpacingDirection = {
  base?: string    // 'none' | 'xs' | 'sm' | 'md' | 'lg' | 'xl' | '2xl' | 'custom'
  custom?: number  // pixel value when base === 'custom'
  sm?: string      // responsive override
  md?: string
  lg?: string
  xl?: string
}

type SpacingGroup = {
  top?: SpacingDirection
  right?: SpacingDirection
  bottom?: SpacingDirection
  left?: SpacingDirection
}

type PresetWithCustom = {
  base?: string
  custom?: number | string
  sm?: string
  md?: string
  lg?: string
  xl?: string
}

type ColorValue = {
  preset?: string   // 'none' | 'primary' | 'secondary' | ... | 'custom'
  custom?: string   // hex or rgb value when preset === 'custom'
  sm?: string
  md?: string
  lg?: string
  xl?: string
}

type StylesData = {
  padding?: SpacingGroup
  margin?: SpacingGroup
  borderRadius?: PresetWithCustom
  borderWidth?: PresetWithCustom
  textSize?: PresetWithCustom
  backgroundColor?: ColorValue
  textColor?: ColorValue
  customCSS?: {
    classes?: string
    inlineCSS?: string
  }
}
```

**2. Create `apps/starter/src/components/admin/StylesPanel.tsx`:**

A `'use client'` React component using standard function declaration (not arrow function).

Use `useField<StylesData>({ path })` from `@payloadcms/ui` where `path` comes from the component props. The field stores and retrieves the entire styles JSON object.

**Component structure:**

A. **Bounding Box UI (always visible, not collapsible):**
   - Visual nested box diagram: outer box = margin (labeled "Margin"), inner box = padding (labeled "Padding"), center shows "Content"
   - Each side (top, right, bottom, left) has a small number input positioned at that side of the box
   - Inputs edit the `base` value for that direction — use a `<select>` dropdown with spacing presets (none, xs, sm, md, lg, xl, 2xl)
   - Layout: Use CSS (in the SCSS file) to position inputs around the boxes — the outer margin box has inputs on its 4 edges, the inner padding box has inputs on its 4 edges
   - Colors: margin area = light orange/amber background, padding area = light green background, content center = light blue — matching Webflow/browser DevTools conventions

B. **Collapsible groups (all collapsed by default):**

   Each group uses a `<details>` element (or a simple div with toggle state) with summary/header:

   i. **Border** — Two rows:
      - Border Radius: `<select>` with borderRadiusPresets for `base`, plus a number input for custom value (shown when base === 'custom')
      - Border Width: `<select>` with borderWidthPresets for `base`, plus a number input for custom value

   ii. **Typography** — One row:
      - Text Size: `<select>` with textSizePresets for `base`, plus a text input for custom value

   iii. **Colors** — Two rows:
      - Background Color: `<select>` with colorPresets for `preset`, plus a color/text input for custom hex/rgb (shown when preset === 'custom')
      - Text Color: same pattern

   iv. **Custom CSS** — Two fields:
      - Extra Classes: text input for `customCSS.classes`
      - Inline CSS: text input for `customCSS.inlineCSS`

**State management:**
- Read initial value from `useField` hook's `value`
- On any input change, create a new styles object with the updated value and call `setValue(newStyles)`
- Use a helper function `updateNestedValue(styles, path, value)` to immutably update deeply nested properties (e.g., `['padding', 'top', 'base']` -> 'md')
- For the spacing bounding box, when a select changes from a non-'none' value to 'none', clear that direction's data

**Important implementation notes:**
- Use standard function declarations for all components (per CLAUDE.md)
- Use early returns, not nested ifs
- Use `size-` classes instead of `h-/w-` when dimensions are the same
- Keep responsive overrides OUT of the visual UI for now (base values only) — responsive is an advanced feature that can be added later. The JSON schema supports it but the UI only edits `base` values initially.
- Import SCSS file at the top of the component
